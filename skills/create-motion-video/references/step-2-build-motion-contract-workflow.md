# Step 2 Workflow: Build Motion Contract

## Objective

Convert the approved storyboard into an implementation contract for pacing, motion, visual language, and embed placement.

## Required actions

1. Expand the approved storyboard into scene-level timing and motion behavior with [scene-plan-template.md](../assets/templates/scene-plan-template.md).
2. Define composition format, scene durations, transitions, captions, and motion primitives with [motion-contract-template.md](../assets/templates/motion-contract-template.md).
3. Apply the blog-specific visual rules from [blog-design-alignment.md](blog-design-alignment.md).
4. Apply the accessibility rules from [accessibility-checklist.md](accessibility-checklist.md).
5. Apply the performance rules from [performance-guardrails.md](performance-guardrails.md).
6. Define the article embed location, deterministic final filename, and preview outputs before implementation starts.

## Done when

- The scene contract is explicit.
- Visual and motion rules are explicit.
- Embed location and filename are explicit.
- Preview outputs are explicit.
